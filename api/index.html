API¶

Primitives¶

Pulsar is built on top of a set of primitive classes which handle the different aspects of the concurrent framework. These primitive classes are:

  • EventLoop: handles scheduling of asynchronous and recurrent events. Designed to conform with pep-3156 eventloop interface.
  • Deferred: handles asynchronous call backs. Designed along the lines of futures or promises, with a similar implementation of twisted deferred.
  • Actor: manage parallel execution in threads or processes. Each live actor has its own EventLoop.
  • Transport: abstract class handling end-to-end communication services for applications. Designed to conform with pep-3156 transport interface.
  • ProtocolConsumer: base class for consuming stream of data provided by the Transport via a Server or Client Connection.

Framework¶

A second layer of classes, forming the concurrent framework, is built directly on top of pulsar primitives. These framework classes are:

  • Arbiter manages the execution of pulsar-powered applications.
  • apps.Application is the base class for all higher level pulsar usage.
