Protocols API¶

This part of the pulsar API is about classes responsible for implementing the Protocol/Transport paradigm as well as the Server, the base class for all socket servers.

Transport¶

class pulsar.Transport(event_loop, sock, protocol, max_buffer_size=None, read_chunk_size=None, timeout=None)¶

Base class for pulsar transports. Design to conform with pep-3156 as close as possible until it is finalised. A transport is an abstraction on top of a socket or something similar. Form pep-3153:

Transports talk to two things: the other side of the connection on one hand, and a protocol on the other. It’s a bridge between the specific underlying transfer mechanism and the protocol. Its job can be described as allowing the protocol to just send and receive bytes, taking care of all of the magic that needs to happen to those bytes to be eventually sent across the wire.

The primary feature of a transport is sending bytes to a protocol and receiving bytes from the underlying protocol. Writing to the transport is done using the write() and writelines() methods. The latter method is a performance optimisation, to allow software to take advantage of specific capabilities in some transport mechanisms.

eventloop¶

The EventLoop for this Transport.

protocol¶

The Protocol for this Transport.

sock¶

the socket/pipe for this Transport.

connecting¶

True if the transport is connecting with remote server.

writing¶

True if the transport has data in its writing buffer.

closing¶

True if the transport is about to close.

closed¶

True if the transport is closed.

write(data)¶

Write some data bytes to the transport. This does not block; it buffers the data and arranges for it to be sent out asynchronously.

writelines(list_of_data)¶

Write a list (or any iterable) of data bytes to the transport. If list_of_data is a generator, and during iteration an empty byte is yielded, the function will postpone writing the remaining of the generator at the next loop in the eventloop.

pause()¶

A Transport can be paused and resumed. Invoking this method will cause the transport to buffer data coming from protocols but not sending it to the protocol. In other words, no data will be passed to the Protocol.data_received() method until resume() is called.

resume()¶

Resume the receiving end. Data received will once again be passed to the Protocol.data_received() method.

close(async=True)¶

Closes the transport.

Buffered data will be flushed asynchronously. No more data will be received. After all buffered data is flushed, the protocol’s connection_lost() method will (eventually) called with None as its argument.

abort()¶

Closes the transport immediately.

Buffered data will be lost. No more data will be received. The protocol’s connection_lost() method will (eventually) be called with None as its argument.

connect()¶

Connect this Transport to a remote server and returns self.

Protocol¶

class pulsar.Protocol(address)¶

Pulsar Protocol conforming with pep-3156. It can be used for both client and server sockets.

  • a client protocol is for clients connecting to a remote server.
  • a server protocol is for socket created from an accept on a Server.
address¶

Address of the client, if this is a server, or of the remote server if this is a client.

transport¶

The Transport for this Protocol. This is obtained once the connection_made() is invoked.

on_connection¶

a Deferred called once the transport is connected.

on_connection_lost¶

a Deferred called once the transport loses the connection with the endpoint.

METHODS

connection_made(transport)¶

Called when a connection is made. The argument is the Transport representing the connection. To send data, call its Transport.write() or Transport.writelines() method. To receive data, wait for data_received() calls. When the connection is closed, connection_lost() is called.

data_received(data)¶

Called by the transport when data is received. By default it feeds the data, a bytes object, into the current_response attribute.

eof_received()¶

Called when the other end calls write_eof() or equivalent.

connection_lost(exc)¶

Called when the connection is lost or closed.

The argument is an exception object or None (the latter meaning a regular EOF is received or the connection was aborted or closed).

set_response(response)¶

Set a new response instance on this protocol. If a response is already available it raises an exception.

Protocol Consumer¶

class pulsar.ProtocolConsumer(connection)¶

A Protocol consumer is responsible for parsing incoming data and, producing no more than one response.

connection¶

The Connection of this consumer

protocol¶

The Protocol of this consumer

transport¶

The Transport of this consumer

feed(data)¶

Feed new data into this ProtocolConsumer. This method must be implemented by subclasses.

finished(result=<object object at 0x404947a0>)¶

Call this method when done with this ProtocolConsumer. By default it calls the Connection.finished() method of the connection attribute.

write(data)¶

Proxy of Transport.write() method of transport.

writelines(lines)¶

Proxy of Transport.writelines() method of transport.

Producer¶

class pulsar.Producer(max_connections=0, timeout=0, connection_factory=None)¶

A Producer of Connection with remote servers or clients. It is the base class for both Server and ConnectionPool. The main method in this class is new_connection() where a new Connection is created and added to the set of concurrent_connections.

concurrent_connections¶

Number of concurrent active connections

received¶

Total number of received connections

timeout¶

number of seconds to keep alive an idle connection

max_connections¶

Maximum number of connections allowed. A value of 0 (default) means no limit.

new_connection(protocol, response_factory, producer=None)¶

‘Called when a new connection is created

close_connections(connection=None)¶

Close connection if specified, otherwise close all active connections.

Connection¶

class pulsar.Connection(protocol, producer, session, response_factory)¶

A client or server connection. It contains the Protocol, the transport producer (Server or Client) and a session number.

protocol¶

The Protocol of this connection

producer¶

The producer of this Connection, It is either a Server or a client Client.

response_factory¶

A factory of ProtocolConsumer instances for this Protocol

session¶

Connection session number. Created by the producer

processed¶

Number of separate requests processed by this connection.

current_response¶

The Consumer currently handling incoming data.

upgrade(response_factory)¶

Update the response_factory attribute with a new ProtocolResponse. This function can be used when the protocol specification changes during a response (an example is a WebSocket response).

finished(response, result=<object object at 0x404947a0>)¶

Call this method with the current response to close the current consumer.

Clients¶

Client¶

class pulsar.Client(timeout=None, client_version=None, max_connections=None, trust_env=True, stream=None, **params)¶

A client for a remote server which handles one or more ConnectionPool of asynchronous connections.

timeout¶

timeout in seconds for an idle connection to be dropped. A value of zero means no timeout.

connection_pool¶

Factory of ConnectionPool.

response_factory¶

Factory of response instances

setup(**params)¶

Setup the client. By default it does nothing.

request(*args, **params)¶

Create a request and invoke the response() method. Must be implemented by subclasses.

response(request, consumer=None)¶

Once a request object has been constructed, the request() method should invoke this method to start the response dance.

Parameters:
  • request – A custom request for the Client
  • consumer – An optional consumer of streaming data, it is the ClientProtocolConsumer.consumer attribute of the consumer returned by this method.
Return type:

An ClientProtocolConsumer obtained form response_factory.

Client Connection Pool¶

class pulsar.ConnectionPool(request, **params)¶

A Producer of of active connections for client protocols. It maintains a live set of connections.

address¶

Address to connect to

classmethod get(request, pools, **params)¶

Build a new ConnectionPool if not already available in pools dictionary.

release(connection)¶

Releases the connection back to the pool

remove(connection)¶

Remove the connection from the pool

get_or_create_connection(client)¶

Get or create a new connection for client

Client Protocol Consumer¶

class pulsar.ClientProtocolConsumer(connection, request, consumer)¶

A ProtocolConsumer for a Client.

request¶

The Request instance.

consumer¶

Optional callable which can be used by the feed method to send data to another listening consumer. This can be used to stream data as it arrives (for example).

begin()¶

Connect, send data and wait for results.

feed(data)¶

Called when new data has arrived from the remote server.

send(*args)¶

Write the request to the server. Must be implemented by subclasses. This method is invoked by the begin() method once the connection is established. Not called directly.

Servers¶

Server¶

class pulsar.Server(event_loop, sock, protocol_factory=None, timeout=None, max_connections=0, response_factory=None, connection_factory=None)¶

A Producer for all server’s listening for connections on a socket. It is a producer of Transport for server protocols.

protocol_factory¶

A factory producing the Protocol for a socket created from a connection of a remote client with this server. This is a function or a Protocol class which accept two arguments, the client address and the response_factory attribute. This attribute is used in the create_connection() method.

response_factory¶

Optional callable or ProtocolResponse class which can be used to override the Protocol.response_factory attribute.

event_loop¶

The EventLoop running the server.

address¶

Server address, where clients send requests to.

on_close¶

A Deferred called once the Server is closed.

create_connection(sock, address)¶

Create a new server Protocol ready to serve its client.

close()¶

Close the server

ready_read()¶

Callback when a new connection is waiting to be served. This must be implemented by subclasses.

classmethod create(eventloop=None, sock=None, address=None, backlog=1024, name=None, close_event_loop=None, **kw)¶

Create a new server!
