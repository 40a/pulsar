Writing Clients¶

The code for this example is located in the examples.echo.manage module.

Pulsar provides several classes for writing clients. The first step is to subclass pulsar.ProtocolConsumer which is needed for two reasons:

  • It encodes and sends the request to the remote server.
  • It listen (if needed) for incoming data from the remote server.

There are two methods which needs implementing:

  • pulsar.ProtocolConsumer.start_request() to kick start a request to a remote server.
  • pulsar.Protocol.data_received(), invoked by the pulsar.Connection when new data has been received from the remote server.

In this example we implement a very simple Echo server/client pair. Because this example has symmetric client and server protocols, the EchoProtocol will also be used for the server.

Echo Protocol¶

class examples.echo.manage.EchoProtocol(connection=None)[source]¶

Echo protocol consumer for client and servers.

separator¶

A separator for messages.

data_received(data)[source]¶

Implements the pulsar.Protocol.data_received() method. It simple search for the separator and if found it invokes the response() method with the value of the message.

response(data)[source]¶

Clients store the message in the result attribute, while servers sends the message back to the client.

Run The example¶

To run the server:

python manage.py

Open a new shell, in this directory, launch python and type:

>>> from manage import Echo
>>> p = Echo('localhost:8060')
>>> response = p.request('Hello')
