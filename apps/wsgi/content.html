Asynchronous Content¶

The pulsar.apps.wsgi.content introduces several utility classes handling asynchronous content on a WSGI server. The main class of this module is the AsyncString. Web framework are smart ways of concatenating bytes which need to be displayed on a web page or returned as a response to and HTTP client.

Return the StreamRenderer for this instance. This method can be called once only since it invokes the stream() method.

stream(request)[source]¶

This is the most important method of an AsyncString. It is called by content() or by the parent of this AsyncString. It returns an iterable (list, tuple, generator) over strings, that means unicode/str for python 2, and str for python 3 or asynchronous elements which result in strings. This method can be called once only, otherwise a RuntimeError occurs.

Hook called if request is available, just before the do_stream() is executed.

Return a, possibly, asynchronous WSGI iterable. This method asynchronously wait for content() and subsequently starts the wsgi response.

do_stream(request)[source]¶

Perform the actual streaming of this AsyncString. This method can be re-implemented by subclasses.

to_string(stream)[source]¶

Once the StreamRenderer, returned by content() method, is ready, meaning it has no more asynchronous elements, this method get called to transform the stream into the content string. This method can be overwritten by derived classes.

Parameters:stream – a collections containing strings/bytes used to build the final string/bytes.
Returns:a string or bytes
render(request=None)[source]¶

A shortcut function for synchronously rendering a Content. This is useful during testing. It is the synchronous equivalent of content().

Asynchronous Html¶

class pulsar.apps.wsgi.content.Html(tag, *children, **params)[source]¶

An AsyncString for html elements. The AsyncString.content_type attribute is set to text/html.

Parameters:
  • tag – Set the tag attribute. Must be given and can be None.
  • children – Optional children which will be added via the AsyncString.append() method.
  • params – Optional keyed-value parameters.

Special (optional) parameters:

  • cn class name or list of class names.
  • attr dictionary of attributes to add.
  • data dictionary of data to add (rendered as HTML data).
  • type type of element, only supported for tags which accept the type attribute (for example the input tag).

Any other keyed-value parameter will be added as attribute, if in the set of available_attributes or as data().

A container of Html meta tags. To add new meta tags use the add_meta() method rather than accessing the meta attribute directly.

A container of css links. Rendered just after the meta container.

scripts¶

A container of Javascript files to render at the end of the body tag. To add new javascript files simply use the append method on this attribute. You can add relative paths:

html.head.scripts.append('/media/js/scripts.js')

as well as absolute paths:

html.head.scripts.append('https://ajax.googleapis.com/ajax/libs/jquery/1.7.2/jquery.min.js')
