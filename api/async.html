Asynchonous API¶

Event loop¶

Timed Call¶

class pulsar.TimedCall(deadline, callback, args)¶

An EventLoop callback handler. This is not initialised directly, instead it is created by EventLoop.call_soon(), EventLoop.call_later(), EventLoop.call_soon_threadsafe() and so forth.

deadline¶

a time in the future or None.

callback¶

The callback to execute in the eventloop

cancelled¶

Flag indicating this callback is cancelled.

cancel()¶

Attempt to cancel the callback.

EventLoop¶

class pulsar.EventLoop(io=None, logger=None, poll_timeout=None, timer=None, iothreadloop=True)¶

A pluggable event loop which conforms with the pep-3156 API. The event loop is the place where most asynchronous operations are carried out.

io¶

The I/O implementation. If not supplied, the best possible implementation available will be used. On posix system this is epoll, or else select. It can be any other custom implementation as long as it has an epoll like interface.

cpubound¶

If True this is a CPU bound event loop, otherwise it is an I/O event loop. CPU bound loops can block the loop for considerable amount of time.

num_loops¶

Total number of loops

poll_timeout¶

The timeout in seconds when polling with epol or select.

Default: 0.5

tid¶

The thread id where the eventloop is running

iothreadloop¶

True if this EventLoop install itself as the event loop of the thread where it is run.

run()¶

Run the event loop until nothing left to do or stop() called.

run_forever()¶

Run the event loop forever.

run_once(timeout=None)¶

Run through all callbacks and all I/O polls once.

Calling stop() will break out of this too.

run_until_complete(future, timeout=None)¶

Run the event loop until a Deferred future is done. Return the future’s result, or raise its exception. If timeout is not None, run it for at most that long; if the future is still not done, raise TimeoutError (but don’t cancel the future).

stop()¶

Stop the loop after the current event loop iteration is complete

call_at(when, callback, *args)¶

Arrange for a callback to be called at a given time when in the future. The time is an absolute time, for relative time check the call_later() method. Returns a TimedCall with a TimedCall.cancel() method that can be used to cancel the call.

call_later(seconds, callback, *args)¶

Arrange for a callback to be called at a given time in the future. Returns a TimedCall with a TimedCall.cancel() method that can be used to cancel the call. The delay can be an int or float, expressed in seconds. It is always a relative time.

Each callback will be called exactly once. If two callbacks are scheduled for exactly the same time, it is undefined which will be called first.

Callbacks scheduled in the past are passed on to call_soon() method, so these will be called in the order in which they were registered rather than by time due. This is so you can’t cheat and insert yourself at the front of the ready queue by using a negative time.

Any positional arguments after the callback will be passed to the callback when it is called.

call_soon(callback, *args)¶

Equivalent to self.call_later(0, callback, *args).

call_soon_threadsafe(callback, *args)¶

Calls the given callback on the next I/O loop iteration. It is safe to call this method from any thread at any time. Note that this is the only method in EventLoop that makes this guarantee. all other interaction with the EventLoop must be done from that EventLoop‘s thread. It may be used to transfer control from other threads to the EventLoop’s thread.

run_in_executor(executor, callback, *args)¶

Arrange to call callback(*args) in an executor. NOT YET SUPPORTED.

add_reader(fd, callback, *args)¶

Add a reader callback. Return a Handler instance.

add_writer(fd, callback, *args)¶

Add a reader callback. Return a Handler instance.

remove_reader(fd)¶

Cancels the current read callback for file descriptor fd, if one is set. A no-op if no callback is currently set for the file descriptor.

remove_writer(fd)¶

Cancels the current write callback for file descriptor fd, if one is set. A no-op if no callback is currently set for the file descriptor.

add_signal_handler(sig, callback, *args)¶

Whenever signal sig is received, arrange for callback(*args) to be called. Returns a TimedCall handler which can be used to cancel the signal callback.

remove_signal_handler(sig)¶

Remove the signal sig if it was installed and reinstal the default signal handler signal.SIG_DFL.

create_server(**kwargs)¶

Create a new Server.

wake()¶

Wake up the eventloop.

call_repeatedly(interval, callback, *args)¶

Call a callback every interval seconds. It handles asynchronous results. If an error occur in the callback, the chain is broken and the callback won’t be called anymore.

call_every(callback, *args)¶

Same as :meth:call_repeatedly with the only difference that the callback is scheduled at every loop.

call_now_threadsafe(callback, *args)¶

Same as call_soon_threadsafe() with the only exception that if we are calling from the same thread of execution as this EventLoop, the callback is called immediately.

Async Discovery Functions¶

This section describes the asynchronous discover functions which are used throughout the library to access if objects are asynchronous or not. There are two important functions: maybe_async() and maybe_failure() for asynchronous exceptions.

Maybe Async¶

pulsar.maybe_async(value, description=None, event_loop=None, max_errors=0, timeout=None, get_result=True)¶

Return an asynchronous instance only if value is a generator or already an asynchronous instance. If value is asynchronous it checks if it done. In this case it returns the result.

Parameters:
  • value – the value to convert to an asynchronous instance if it needs to.
  • description – optional description, rarely used.
  • event_loop – optional EventLoop.
  • max_errors – The maximum number of exceptions allowed before stopping the generator and raise exceptions. By default it is 0. Pass None to continue the processing of the coroutine regardless of any exception.
  • timeout – optional timeout after which any asynchronous element get a cancellation.
  • get_result – optional flag indicating if to get the result in case the return values is a Deferred already done. Default: True.
Returns:

a Deferred or a Failure or a synchronous value.

Maybe Failure¶

pulsar.maybe_failure(value, msg=None)¶

Convert value into a Failure if it is a stack trace or an exception, otherwise returns value.

Parameters:
  • value – the value to convert to a Failure instance if it needs to.
  • msg – Optional message to display in the log if value is a failure.
Returns:

a Failure or the original value.

Set Async¶

pulsar.set_async(maybe_async_callable, maybe_failure_callable)¶

Set the asynchronous and failure discovery functions. This can be used when third-party asynchronous objects are used in conjunction with pulsar Deferred and Failure.

Is async¶

pulsar.is_async(obj)¶

Return True if obj is an asynchronous object

Is failure¶

pulsar.is_failure(obj, classes=None)¶

Check if obj is a Failure. If optional classes are given, it checks if the error is an instance of those classes.

Async Utilities¶

Async¶

class pulsar.async(**params)¶

A decorator class which invokes maybe_async() on the return value of the function it is decorating. The input parameters and the outcome are the same as maybe_async().

Typical usage:

@async()
def myfunction(...):
    ...

It can also be used to safely call functions:

async()(myfunction, ...)

This syntax will always return a Deferred:

async(get_result=False)(myfunction, ...)

Multi Async¶

pulsar.multi_async(iterable, **kwargs)¶

This is an utility function to convert an iterable into a MultiDeferred element.

Safe Async¶

pulsar.safe_async(self, callable, *args, **kwargs)¶

Safely execute a callable and always return a Deferred, even if the callable is not asynchronous. Never throws.

Async Sleep¶

pulsar.async_sleep(timeout)¶

The asynchronous equivalent of time.sleep(timeout). Use this function within a coroutine when you need to resume the coroutine after timeout seconds. For example:

...
yield async_sleep(2)
...

Asynchronous Classes¶

While Actor represents the concurrent side of pulsar, the Deferred adds the asynchronous flavour to it by using callbacks functions similar to twisted.

Deferred¶

class pulsar.Deferred(description=None, timeout=None, event_loop=None)¶

The main class of pulsar asynchronous engine. It is a callback which will be put off until later. It conforms with the tulip.Future interface with an implementation similar to the twisted.defer.Deferred class.

Parameters:
  • description – optional description which set the description attribute.
  • timeout – optional timeout. If greater than zero the deferred will be cancelled after timeout seconds if no result is available.
  • event_loop – If supplied, it is the EventLoop associated with this Deferred. If not supplied, the default event loop is used. The event loop associated with this Deferred is accessed via the event_loop attribute.
description¶

An optional description of the Deferred (for debugging and representation purposes).

paused¶

Integer indicating the number of times this Deferred has been paused because the result of a callback was another Deferred.

result¶

This is available once the Deferred is done. Note, this can be anything, including another Deferred. Trying to access this attribute when done() is False will result in an AttributeError exception.

event_loop¶

The EventLoop associated with this Deferred. It can be set during initialisation.

timeout¶

The TimedCall which handles the timeout of this Deferred. Available only when a timeout is set.

set_timeout(timeout)¶

Set a the timeout for this Deferred. It returns self so that other methods can be concatenated.

cancelled()¶

pep-3156 API method, it returns True if the Deferred was cancelled.

done()¶

pep-3156 API method, it returns True if the Deferred is done, that is it was called or cancelled.

cancel(msg='')¶

pep-3156 API method, it cancel the deferred and schedule callbacks. If the deferred is waiting for another Deferred, forward the cancellation to that one. If the Deferred is already done(), it does nothing.

Parameters:msg – Optional message to pass to the when the CancelledError is initialised.
running()¶

pep-3156 API method, always returns False.

add_done_callback(fn)¶

pep-3156 API method, Add a callback to be run when the Deferred becomes done. The callback is called with a single argument, the Deferred object.

set_result(result)¶

pep-3156 API method, same as callback()

set_exception(exc)¶

pep-3156 API method, same as callback()

add_callback(callback, errback=None, continuation=None)¶

Add a callback as a callable function. The function takes at most one argument, the result passed to the callback() method. If the errback callable is provided it will be called when an exception occurs.

add_errback(errback, continuation=None)¶

Same as add_callback() but only for errors.

add_both(callback, continuation=None)¶

Equivalent to self.add_callback(callback, callback).

callback(result=None, state=None)¶

Run registered callbacks with the given result. This can only be run once. Later calls to this will raise InvalidStateError. If further callbacks are added after this point, add_callback() will run the callbacks immediately.

Returns:the result input parameter
raise_all()¶

raise an exception only if done() is True and the result is a Failure

Multi Deferred¶

class pulsar.MultiDeferred(data=None, type=None, raise_on_error=True, handle_value=None, log_failure=False, **kwargs)¶

A Deferred for managing a collection of independent asynchronous objects. The collection can be either a list or a dict. The MultiDeferred is recursive on values which are generators, Mapping, lists and sets. It is not recursive on immutable containers such as tuple, and frozenset.

locked¶

When True, the update() or append() methods can no longer be used.

type¶

The type of multi-deferred. Either a list or a Mapping.

raise_on_error¶

When True and at least one value of the result collections is a Failure, the callback will receive the failure rather than the collection of results.

Default True.

lock()¶

Lock the MultiDeferred so that no new items can be added. If it was alread locked a runtime exception occurs.

update(stream)¶

Update the MultiDeferred with new data. It works for both list and dict type.

append(value)¶

Append a new value to this asynchronous container. Works for a list type only.

Task¶

class pulsar.Task(gen, max_errors=0, event_loop=None, **kwargs)¶

A Deferred coroutine is a consumer of, possibly, asynchronous objects. The callback will occur once the coroutine has stopped (when it raises StopIteration), or a preset maximum number of errors (default 1) has occurred. Instances of Task are never initialised directly, they are created by the maybe_async() function when a generator is passed as argument.

Failure¶

class pulsar.Failure(err=None, msg=None)¶

The asynchronous equivalent of python Exception. It accumulate errors during Deferred callbacks.

traces¶

List of (errorType, errvalue, traceback) occured during the execution of a Deferred. Usually, this list contains one element only.

logged¶

Check if the error was logged. It can be a way of switching off logging for certain errors.

error¶

Last python Exception instance if available.

append(failure)¶

Append new failure to self. The input failure can be another Failure, an Exception or a system exception info tuple obtained from sys.exc_info().

clear()¶

Clear this Failure

isinstance(classes)¶

Check if error is an instance of exception classes.

Events Handler¶

class pulsar.EventHandler¶

A Mixin for handling one time events and events that occur several times. This mixin is used in Protocol and Producer for scheduling connections and requests.

ONE_TIME_EVENTS = ()¶

Event names which occur once only. Implemented as Deferred.

MANY_TIMES_EVENTS = ()¶

Event names which occur several times. Implemented as list of callables.

event(name)¶

Return the handler for event name.

bind_event(event, callback)¶

Register a callback with event. The callback must be a callable which accept one argument only.

fire_event(event, event_data=<object object at 0x10b342170>, sender=None)¶

Dispatches event_data to the event listeners. * If event is a one-time event, it makes sure that it was not fired before.

Parameters:
  • event_data – if not provided, self will be dispatched instead, if an error it will be converted to a Failure.
  • sender – optional sender of this event. It is only used for dispatching global events.
Returns:

boolean indicating if the event was fired or not.
