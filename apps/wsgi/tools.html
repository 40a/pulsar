Utilities¶

HTTP Parsers¶

Pulsar ships with its own HTTP parser used by both server and the HTTP client. Headers are collected using the Headers data structure which exposes a list/dictionary-type interface.

At runtime, pulsar checks if the http-parser package and cython are available. If this is the case, it switches the default HTTP parser to be the one provided by the http_parser.paser module. To check if the C parser is the default parser:

from pulsar.lib import hasextensions

Authentication¶

Utility classes for managing Header Authentication, used both by servers and clients.

header parser¶

pulsar.apps.wsgi.plugins.parse_authorization_header(value, charset='utf-8')[source]¶

Parse an HTTP basic/digest authorization header transmitted by the web browser. The return value is either None if the header was invalid or not given, otherwise an Auth object.

Parameters:value – the authorization header to parse.
Returns:a Auth or None.
