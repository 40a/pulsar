Asynchronous Content¶

Utility classes for Asynchronous Http body strings.

Asynchronous String¶

class pulsar.apps.wsgi.content.AsyncString(*children)[source]¶

Class for asynchronous strings which can be used with pulsar WSGI servers.

content_type¶

Content type for this AsyncString

encoding¶

Charset encoding for this AsyncString

content(request=None)[source]¶

Return the StreamRenderer for this instance. This method can be called once only.

stream(request)[source]¶

Return an iterable over strings, that means unicode/str for python 2, and str for python 3. This method can be called once only otherwise a RuntimeError occurs.

http_response(request)[source]¶

Return the WSGI iterable. The iterable yields empty bytes untill the AsyncString has its result ready. Once ready it sets its value as the content of a WsgiResponse

render(request=None)[source]¶

A shortcut function for synchronously rendering a Content. This is useful during testing.

to_string(stream)¶

Once the stream is ready (no more asynchronous elements) this functions get called to transform the stream into a string. This method can be overwritten by derived classes.

Parameters:stream – a collections containing data used to build the string.
Returns:a string or bytes

add_to_html(request)[source]¶

The request holds a reference to the Html document being rendered. This function can be used to add media files or response headers. For example:

request.html_documnet.head.scripts.append('http://...')
request.response.headers['ETag'] = ...

By default it does nothing.
